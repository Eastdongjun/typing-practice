# 需求资料目录

## 目录结构

| 目录 | 用途 |
|---|---|
| `00-intake` | 原始需求、会议纪要、业务问题、附件索引 |
| `01-background` | 项目背景、目标、范围、术语和干系人 |
| `02-as-is` | 现状流程、问题清单、现有系统和数据 |
| `03-to-be` | 目标流程、未来业务模式和总体方案 |
| `04-prd` | 正式产品需求文档、功能清单和权限矩阵 |
| `05-prototype` | 页面清单、页面结构、字段和交互说明 |
| `06-data` | 数据对象、数据字典、状态机和指标口径 |
| `07-integration` | 接口、消息、文件交换和外部系统集成 |
| `08-acceptance` | 验收标准、测试场景和追踪矩阵 |
| `09-decisions` | 决策、假设、待确认问题和变更记录 |
| `templates` | 标准模板 |

## 文件命名规则

建议：`YYYYMMDD-主题-文档类型-v版本.md`

示例：

- `20260629-访客预约-需求受理-v0.1.md`
- `20260629-访客预约-PRD-v0.1.md`
- `20260629-访客预约-验收标准-v0.1.md`

## 需求编号规则

`模块缩写-类型-四位序号`

- 功能需求：`ATT-FR-0001`
- 业务规则：`ATT-BR-0001`
- 非功能需求：`ATT-NFR-0001`
- 接口需求：`ATT-INT-0001`
- 数据需求：`ATT-DATA-0001`

## 文档状态

`草稿 → 待评审 → 已批准 → 已实现 → 已验收 → 已归档`
