# 项目上下文 — 打字练习

## 1. 项目基本信息

| 项目项 | 内容 |
|---|---|
| 项目名称 | 打字练习 (Typing Practice) |
| 项目负责人 | Dongjun |
| GitHub 仓库 | https://github.com/Eastdongjun/typing-practice |
| 默认分支 | main |
| 当前版本 | V1.0.0 |

## 2. 建设背景

- **当前问题**：缺少一个桌面端的指法训练工具，特别是面向中国学生的古诗文打字练习
- **建设原因**：将打字训练与古诗文学习结合，同时提供家长监控和教师管理能力
- **期望价值**：提升学生打字速度和古诗文记忆，家长可追踪学习进度
- **关键成功指标**：学生打字速度提升 30%+，古诗文正确率 90%+

## 3. 业务范围

### 纳入范围
- 5 种打字练习模式（英文词、名言、代码、中文、指法训练）
- 10 套主题皮肤 + 自动深色/浅色切换
- 桌面端自适应（1366px ~ 4K）
- 可视化指法键盘（Mac/Win 布局）
- 实时速度/准确率/进度统计
- 用户角色系统（学生/家长/教师/管理员）
- 古诗文内容库（小学75首 + 初中61首）
- 家长监控看板与统计分析
- 班级管理与作业布置

### 不纳入范围
- 移动端 / 平板
- Web 端（仅浏览器直接打开 HTML 可用，但不作为主推）
- 实时多人对战
- 语音输入

### 外部系统边界
- 无外部系统依赖（纯本地应用 + GitHub 存储）

## 4. 用户角色

| 角色 | 职责 | 可见数据范围 | 关键操作 |
|---|---|---|---|
| 学生(小学) | 练习打字、学古诗 | 自己的成绩 | 打字练习、查看成绩、换皮肤 |
| 学生(中学) | 练习打字、学古诗 | 自己的成绩 | 同小学 |
| 家长 | 查看孩子学习情况 | 绑定孩子的数据 | 查看报告、设置时长限制 |
| 教师 | 管理班级、布置作业 | 所管班级学生数据 | 创建班级、布置作业、查看全班 |
| 管理员 | 系统管理 | 全部数据 | 内容审核、系统配置 |

## 5. 业务术语

| 术语 | 定义 |
|---|---|
| WPM | Words Per Minute，英文打字速度（5字符=1词） |
| CPM | Characters Per Minute，中文打字速度 |
| KPM | Keys Per Minute，指法练习速度 |
| 指法训练 | 针对特定手指管辖按键的专项练习 |
| Phase | 开发阶段划分 |

## 6. 技术约束

- 前端技术：纯 HTML/CSS/JS（零框架依赖）
- 桌面壳：Electron 35
- 构建工具：electron-builder
- 数据库：待定（Phase 2 引入用户系统时确定）
- 部署方式：GitHub Release 分发安装包
- 身份认证：待定（Phase 2）
- 日志审计：待定

## 7. 设计约束

- 页面风格：深色默认，CSS 变量主题系统，现代化极简
- 移动端/PC 端：仅桌面端
- 多组织/多租户：教师-班级-学生三级
- 数据权限：家长仅看自己孩子，教师仅看自己班级
- 字体：等宽字体（SF Mono / Cascadia Code / JetBrains Mono / Consolas）

## 8. 交付物格式

- PRD：Markdown
- 流程图：Mermaid（如需要）
- 数据字典：Markdown 表格
- 接口说明：REST API（Phase 2+）
- 原型：页面清单 + 字段/按钮/状态说明
- 验收标准：Given/When/Then

## 9. 决策原则

1. 零依赖优先：能用纯 HTML/CSS/JS 就不用框架
2. 桌面优先：不做移动端适配
3. 安全第一：Electron 必须 sandbox + contextIsolation
4. 编码正确：所有文件 UTF-8，传输必须 base64 防乱码
5. 先验证再提交：本地跑通再推 GitHub
